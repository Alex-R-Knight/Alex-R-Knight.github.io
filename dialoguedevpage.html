Mechanic - Branching Dialogue System:

Initial Foundations:

I wanted to make a dialogue system using proximity-based intercoms, so the player could interact with the environment and speak at the same time.
Whilst all visuals are placeholder, I have completed the functional side of the first iteration of this system.

Dialogue data is held in a data table. The intercom retrieves a table entry, which has the text along with all options, their text and the data table addresses of their following entries.

The UI uses typewriter-style movement to display text and also to highlight a player response when it is chosen, using Rich Text widget elements.

The intercom terminates dialogue if the player moves too far away. To make this more intuitive, signal lights were added.
One warns when the player is close to exiting the area of use, whilst the other indicates use and flashes to signal incoming messages.
Because intercoms in a level would be connected, the gamemode stores and updates a "bookmark" table address from which to start dialogue, updated by other events in-game or if a conversation is ended prematurely.
